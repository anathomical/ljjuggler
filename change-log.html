LJ Juggler

A Google Chrome extension for switching between user accounts on LiveJournal.

http://rushin-doll.net/lj-juggler/

As always, feel free to send questions, concerns, comments, or suggestions via email to rushin.doll@gmail.com.

Version 3.0.0

User-facing

  • When adding a new account on the options page, users now select which of the supported sites the account is for. (If you know of other sites you'd like to see supported, let us know. They're pretty easy to add.)
  • Modified the highlighting behavior of the UI to show all accounts you are logged into across all services. This can result in multiple accounts listed as "logged in".
  • This is less a change than a not-change, but it's worth mentioning. The login interface does not differentiate between services. All accounts are listed alphabetically. This means that if you use multiple services your accounts will be all mixed together. We're pondering a UI fix on this one, so again if you have suggestions, let us know.

Behind-the-scenes

  • Early-stage LJ clone support. The system now allows users to register accounts for LJ clone sites, not just LiveJournal.com.
  • Complete overhaul of the behind-the-scenes account highlighting logic. It's significantly more efficient, and more accurate now. It also handles highlighting for multiple accounts now.

Version 2.2.0

User-facing

  • When removing an account on the options page, LJ Juggler now asks you to click a confirmation button before deleting it, just in case you accidentally hit the X.
  • When adding a new account on the options page, LJ Juggler confirms that the username and password you provide can actually be used to log in. This helps save you from typos and solves error-checking problems later on.
  • When you add a new account on the options page, LJ Juggler automatically logs you into that account. We're not sure this is a feature, it might well be a bug (let us know!), but it was the easiest way to handle the validation of accounts.

Behind-the-scenes

  • Added a response-code parsing function for login attempts. LJ Juggler can now do more than just fail to log in if something goes wrong, it can try to tell developers why (in the console).
  • Moved version check and update code into background.html. This actually saves a LOT of CPU cycles since previously there was a fairly involved asynchronous version check every time you opened the Juggler window.
  • Some general code clean up that hopefully paves the way for adding support for LJ clone sites.

Version 2.1.2

User-facing

  • Minor UI tweak: modified highlight behavior so that it is slightly more consistent and has slightly higher visibility.
  • Added a feature to make site-scheme selection permanent. In previous versions it was a session-length cookie and would expire (and need to be reset) when your browser closed. Now if you set one, it'll stick around for a year. Currently the system will use the same sitescheme for ALL accounts. If different schemes for different accounts is really important to you, let me know and I'll see what I can do!
  • Fixed a bug in the cookie expiration calculator so now all cookies set last one year instead of until February of 40572 or some other insane date.

Version 2.1.1

User-facing

  • Tweaked CSS so that the mouse pointer stays as a pointer object rather than converting to a cursor when hovering over the user-option that you are currently logged in as.

Version 2.1.0

User-facing

  • Minor tweak to differentiate between a selected account that is currently highlighted because the user is hovering the mouse over it, and one that is not highlighted but still selected.

Behind-the-scenes

  • Generated a localStorage passing handler so that background.html can handle all localStorage requests from any page in the extension.
  • Used an update check to move all the localStorage data to background.html's store so that it's in one place.
  • Tweaked popup.html and options.html so that they use the localStorage set up in background.html.

Version 2.0.0

User-facing:

  • A pop-up alert to appear whenever the extension updates to a new version.
  • Changed logout procedures so that only the current session is ended. This fixes the long-standing problem that would cause LJ Juggler to interfere with other instances of itself on other machines or with instances of ljlogin on Firefox, or with just about any other login system. They should all play nice now.

Behind-the-scenes:

  • Instituted a background page and moved most of the code to it for efficiency and to allow users to do other things while logins change.
  • Changed password storage to store hashes rather than plaintext for slightly better local security.
  • Shifted to cookie-based login and authentication (the same model used by ljlogin for Firefox). Security bump, much more accurate at recognizing which account is logged in, and noticeable speed increase.
